## 1. 数据规模估算

### 1.1 向量数据

| 维度 | 计算 | 结果 |
|------|------|------|
| 向量维度 | BGE-base | 768 |
| 每条向量大小 | 768 × 4B (float32) | 3KB |
| 文档块大小 | 平均 256 字符 | ~256B 原始文本 |

**估算公式**:
```
单条向量存储 = 向量(3KB) + 文本(512B) + 元数据(256B) ≈ 4KB
```

### 1.2 规模对照表

| 文档数 | 向量条数 | 存储空间 | Milvus 显存 |
|--------|----------|---------|------------|
| 1万 | 10万 | 400MB | ~1GB |
| 10万 | 100万 | 4GB | ~10GB |
| 100万 | 1000万 | 40GB | ~100GB |
| 1000万 | 1亿 | 400GB | ~1TB |
| 1亿 | 10亿 | 4TB | ~10TB |

### 1.3 Embedding 模型显存

| 模型 | 参数量 | 显存需求 |
|------|--------|---------|
| text2vec-base-chinese | 125M | 500MB (FP16) |
| bge-large | 434MB | 1.7GB (FP16) |

## 3. 分片策略

### 3.1 Milvus 分片

```python
# 分片数量建议
shard_num = min(num_nodes * 2, 256)  # 最大 256

# 按文档分片
collection = Collection(
    name="documents",
    shards_num=shard_num  # 建议 = CPU核心数 × 2
)
```

### 3.2 分片 key 选择

| 分片策略 | 适用场景 | 实现方式 |
|----------|---------|--------|
| 按 file_id | 文档隔离 | hash(file_id) % shard_num |
| 按用户 | 多租户 | user_id % shard_num |
| 按时间 | 时序数据 | 时间范围分段 |

### 3.3 冷热分离

```
┌─────────────────────────────────────────┐
│                  热数据                │  ← 30天内 + 常用
│          (SSD / 内存加速)              │
├─────────────────────────────────────────┤
│                  温数据                │  ← 30-90天
│              (SSD 普通)               │
├─────────────────────────────────────────┤
│                  冷数据               │  ← 90天+
│           (HDD / 对象存储)             │
└─────────────────────────────────────────┘
```

## 6. 扩容流程

### 6.1 扩容触发条件

```
建议扩容 when:
- CPU > 80% 持续 10min
- 延迟 P99 > 2s 持续 5min
- 错误率 > 2% 持续 3min
```

### 6.2 扩容步骤

```bash
# 1. 增加 Milvus 节点
docker compose up -d --scale milvus=4

# 2. 增加 Collection 分片
milvus.collection.increment_shards(shard_num=16)

# 3. 验证
curl http://localhost:19530/health
```

### 6.3 缩容流程

```bash
# 非高峰期缩容（成本优化）
docker compose up -d --scale milvus=2
```
