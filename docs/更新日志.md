## v2.1.0 - 2024-05 (生产级加固)

### 1. 连接池修复 - 长连接复用

**维度**: 连接管理
**作用**:
- 保持 Milvus 长连接，不反复创建销毁
- 健康检查 + 自动重连
- 线程安全的单例模式

**解决的问题**:
- 之前每次请求都创建新连接，overhead 大
- 连接断开后无感知
- 并发安全问题

**代码示例**:
```python
class MilvusConnection:
    def ensure_healthy(self):
        if not self.is_healthy():
            self.connect()  # 自动重连
```

### 2. 重试策略 - 指数退避 + 熔断

**维度**: 可靠性
**作用**:
- 指数退避：重试间隔递增 (1s, 2s, 4s...)
- 熔断器：连续失败后 OPEN，拒绝请求
- 防止下游故障时雪崩

**解决的问题**:
- 简单重试没有退避，越失败越快
- 下游挂了还在一直重试
- 无感知，告警不及时

**代码示例**:
```python
class ExponentialBackoff:
    def get_delay(self, attempt):
        return min(self.base * (self.multiplier ** attempt), self.max_delay)

class CircuitBreaker:
    # CLOSED -> OPEN -> HALF_OPEN -> CLOSED
```

### 4. 缓存 - 多级 + 一致性

**维度**: 性能 + 一致性
**作用**:
- L1: 进程内缓存 (dict)
- L2: Redis 缓存
- 版本控制，TTL 失效
- 知识库更新时清缓存

**解决的问题**:
- hash 碰撞风险 -> 用 SHA256
- 知识库更新，缓存不过期
- 缓存穿透/击穿/雪崩

**代码示例**:
```python
class MultiLevelCache:
    def invalidate_knowledge_base():
        return cache.invalidate_pattern("rag:chat:*")
```

### 5. 安全 - 渗透防护

**维度**: 安全
**作用**:
- 文件上传安全 (.exe 拒绝)
- 内容安全检查 (PE header)
- Prompt 注入防护
- SSRF 防护
- Embedding 投毒防护

**解决的问题**:
- 上传恶意文件执行
- Prompt 注入绕过
- 内网服务探测
- 模型投毒

**代码示例**:
```python
class FileSecurity:
    FORBIDDEN_EXTENSIONS = {".exe", ".bat", ".sh", ...}

class PromptInjectionProtection:
    DANGEROUS_PATTERNS = [
        r"ignore previous instructions",
        r"forget everything",
    ]
```
