## 一、项目亮点

### 1.1 核心能力

- ✅ **双轨检索**：向量检索 + 知识图谱融合 (RRF)
- ✅ **文档理解**：支持 .txt/.docx 上传，自动分块、向量化
- ✅ **知识图谱**：LLM 自动提取实体关系，存储于 Neo4j
- ✅ **智能问答**：基于检索结果的 LLM 生成答案
- ✅ **图谱可视化**：ECharts 力导向图展示

### 1.2 工程化特性

| 特性 | 说明 |
|------|------|
| 分层架构 | API / Service / Core 三层分离 |
| 依赖注入 | FastAPI Depends + 单例工厂 |
| 配置中心化 | Pydantic Settings |
| 结构化日志 | JSON 格式 + 多级别 |
| 异常体系 | 自定义异常 + 全局处理 |
| 监控 | Prometheus Metrics |
| 测试 | 单元 + 集成测试 |

### 1.3 生产级特性

| 特性 | 说明 |
|------|------|
| 连接管理 | 长连接复用 + 健康检查 |
| 重试策略 | 指数退避 + 熔断器 |
| 限流 | Redis 滑动窗口分布式限流 |
| 缓存 | 多级缓存 (L1+L2) + 版本控制 |
| 安全 | 渗透防护 + Prompt 注入 |
| 备份恢复 | 灾备方案文档 |

### 1.4 性能优化

| 特性 | 说明 |
|------|------|
| GPU 批处理 | Dynamic Batching |
| 两阶段检索 | 向量粗排 + Reranking 精排 |
| 混合检索 | BM25 + 向量融合 |
| 长文档 | Parent Document Retrieval |
| 流式响应 | Server-Sent Events |

## 二、需求与问题

### 2.1 业务需求

| 需求 | 优先级 | 状态 |
|------|--------|------|
| 文档上传 | P0 | ✅ |
| 向量检索 | P0 | ✅ |
| LLM 问答 | P0 | ✅ |
| 知识图谱 | P1 | ✅ |
| 用户认证 | P1 | ✅ |

### 2.2 遇到的问题

| 问题 | 描述 | 解决方案 |
|------|------|----------|
| 连接泄漏 | 每次请求创建新连接 | 连接池 + 长连接复用 |
| 并发阻塞 | Embedding 是同步调用 | ThreadPoolExecutor |
| 熔断失效 | 下游故障持续重试 | 熔断器 + 指数退避 |
| 限流不准 | 单机限流多副本失效 | Redis 分布式限流 |
| 缓存穿透 | hash 碰撞 + 更新失效 | 多级缓存 + 版本控制 |
| 安全渗透 | 文件上传 + Prompt 注入 | 安全过滤器 |
| GPU 利用低 | 逐条处理 | Dynamic Batching |
| 检索不准 | 向量检索召回单一 | 两阶段 + 混合检索 |

## 三、技术选型

### 3.1 技术栈

| 层级 | 选型 | 原因 |
|------|------|------|
| 后端 | FastAPI | 高性能 + 自动文档 + 依赖注入 |
| 向量库 | Milvus | 国产开源 + 支持十亿级 |
| 图库 | Neo4j | 成熟生态 + Cypher |
| Embedding | BGE | 中文效果好 + 开源 |
| LLM | MiniMax | 国产 + 便宜 + 中文优化 |
| 缓存 | Redis | 分布式限流 + 多级缓存 |
| 部署 | Docker Compose | 一键启动 |
| 监控 | Prometheus | 云原生标配 |

### 3.2 选型考虑

| 技术 | 备选 | 未选原因 |
|------|------|----------|
| Milvus | Pinecone | 国产 + 开源 + 便宜 |
| Neo4j | TuGraph | 生态成熟 |
| MiniMax | OpenAI | 便宜 + 中文优化 |
| FastAPI | Flask | 性能 + 自动文档 |

## 四、实现流程

### 4.1 整体架构

```
用户请求
    │
    ▼
┌─────────────────────────────────────────┐
│            FastAPI API Layer             │
│  (路由 + 认证 + 限流 + 中间件)          │
└────────────────┬────────────────────────┘
                 │
    ┌────────────┴────────────┐
    ▼                         ▼
┌──────────────────┐   ┌──────────────────┐
│  Service Layer   │   │  Service Layer  │
│  RAG Service    │   │  Doc Service    │
└────┬───────────┘   └────┬───────────┘
     │                    │
     ▼                    ▼
┌─────────────────────────────────────────┐
│             Core Layer                  │
│  ┌─────────┐ ┌─────────┐ ┌──────────┐ │
│  │ Milvus  │ │ Neo4j   │ │ MiniMax  │ │
│  └─────────┘ └─────────┘ └──────────┘ │
└─────────────────────────────────────────┘
```

### 4.2 核心流程

```
用户问题
    │
    ▼ 向量化 (BGE)
query_vector
    │
    ├─→ Milvus 向量检索 ──→ Top 50
    │
    ├─→ Neo4j 图谱扩展 ──→ 相关实体
    │
    ├─→ RRF 融合排序 ──→ Top 10
    │
    ├─→ LLM 生成 ──→ 答案
    │
    ▼ 返回
用户
```

### 4.3 数据流

```
文档上传
    │
    ▼ 解析 (python-docx)
text
    │
    ▼ 分块 (TextChunker)
chunks []
    │
    ▼ 向量化 (BGE)
vectors []
    │
    ├─→ Milvus (向量)
    │
    ├─→ Neo4j (实体关系)
    │
    ▼ 完成
```

## 六、面试问题 (从简单到复杂)

### 基础篇 (1-3 年)

**Q1: 介绍一下你的项目？**
> 亮点：双轨检索、知识图谱、生产级改造

**Q2: 向量检索的流程是什么？**
> 分块 → 向量化 → 存入 Milvus → 相似度搜索

**Q3: 什么是 RRF 融合？**
> Reciprocal Rank Fusion，多路检索结果融合排序

**Q4: 文档是怎么分块的？**
> 按段落 + 滑动窗口 + 重叠

**Q5: 遇到过什么性能问题？怎么解决的？**
> GPU 利用率低 → Dynamic Batching

### 中级篇 (3-5 年)

**Q6: 连接池怎么设计的？**
> 长连接复用 + 自动重连 + 健康检查

**Q7: 熔断器怎么实现的？**
> CLOSED → OPEN → HALF_OPEN 状态机

**Q8: 分布式限流怎么做？**
> Redis 滑动窗口 + 多级限流

**Q9: 两阶段检索是什么？**
> 向量粗排 + Cross-Encoder 精排

**Q10: 混合检索的融合公式？**
> score = α × vector + (1-α) × bm25

### 高级篇 (5 年+)

**Q11: 1 亿向量数据怎么分片？**
> 按 file_id hash 分片 + 热点数据分层

**Q12: RAG 召回不完整怎么办？**
> Query 改写 + 多路检索 + 重排序

**Q13: 怎么评估 RAG 效果？**
> 召回率 + 准确率 + 延迟 + ���本

**Q14: 成本怎么优化？**
> 冷热分离 + 按量付费 + Spot 实例

**Q15: 如果 LLM 幻觉怎么办？**
> 信源分级 + 引用校验 + 降级策略
